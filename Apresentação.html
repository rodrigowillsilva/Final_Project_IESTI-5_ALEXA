Project Overview

Objetivo do Projeto

Desenvolvimento e deploy de um sistema completo de Inteligência Artificial Generativa na Borda (Edge AI), capaz de controlar dispositivos físicos e responder a comandos naturais sem dependência de nuvem.


Escopo Realizado

  • Hardware: Configuração e instrumentação da Raspberry Pi 5 com sensores e atuadores.
  • Modelo SLM: Instalação e orquestração do Llama 3.2 (3B parâmetros) quantizado.
  • Integração de Software: Desenvolvimento de pipeline Python modular (Audio → Texto → Inferência → Ação).
  • Funcionalidades: Controle de ambiente (Luz/Temperatura) e Player de Música.

Especificações

  • Unidade de Processamento: Raspberry Pi 5 (8GB RAM).
  • Entrada de Comando: Botão Tátil (GPIO 20) com Pull-up interno.
  • Sensores: DHT22 (Temperatura e Umidade) no GPIO 16 com resistor 4k7Ω.
  • Atuadores:
    • LED Verde (GPIO 26) - Simulação de Lâmpada.
    • LED Amarelo (GPIO 19) - Feedback de Status.

Results

Resultados

  • Viabilidade em Edge: Aplicação funcional em ambiente limitado (fora de grandes clusters).
  • Experiência do Usuário: Integração speech-to-text criando uma experiência similar à "Alexa".
  • Versatilidade: Execução desde controle de hardware (LEDs) e música até busca fundamentada (RAG).

Challanges and Solutions

Desafios

  • Precisão do Function Calling: Variabilidade do Llama 3.2 com muitas funções.
  • Logística de Hardware: Apenas uma placa disponível para o time.
  • Latência de Inferência: Gargalo esperado devido ao hardware limitado (especialmente na detecção de música).

Soluções

  • Solução: Refinamento do System Prompt para reforçar contratos
  • Solução: Desenvolvimento modular e integração posterior.

Future Improvements

Melhorias Futuras

  • Interface Gráfica (GUI): Facilitar o uso para leigos.
  • Conectividade: Controle de dispositivos externos via Bluetooth.
  • Flexibilidade: Suporte a múltiplos "cérebros" (SLMs) para diferentes capacidades de hardware.
  • Otimização: Foco na redução da latência e aumento da precisão

Conclusions

Conclusões

  • Validação Arquitetural: Sucesso do assistente doméstico privacy-first na Raspberry Pi 5.
  • Independência da Nuvem: Processamento semântico e controle local sem exfiltração de dados.
  • Base Sólida: Prova de conceito estabelecida para expansão do ecossistema open source.
